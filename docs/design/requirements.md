# 需求规格说明书

## 1. 项目背景

为教育行业提供AI数字分身解决方案，让教师能够创建自己的智能助教，为学生提供7x24小时的个性化教学服务。

### 1.1 项目概述

- **项目名称**: 数字分身系统 (Digital Twin System)
- **项目版本**: V2.0 (Harness模式全新开发)
- **技术栈**: Go + Gin + Taro + Chroma DB + 大模型API
- **架构模式**: Harness插件化架构（全新项目，无历史迁移负担）

### 1.2 项目目标

- 实现多老师多知识库支持，每位教师拥有独立的数字分身
- 支持学生个性化记忆管理，提供因材施教的学习体验
- 采用Harness插件化架构，提供可插拔的AI能力扩展
- 配置驱动开发，降低运维复杂度，支持快速迭代

## 2. 用户角色

### 2.1 教师 (Producer)
- 知识内容生产者：上传和管理教学文档、知识库
- 学生管理者：查看学生学习进度、对话记录
- 教学数据分析者：分析学生学习情况、优化教学策略
- 数字分身定制者：配置分身的对话风格、教学策略

### 2.2 学生 (Consumer)
- 知识消费者：与数字分身进行苏格拉底式对话学习
- 学习进度被跟踪者：学习行为和进度被系统记录
- 个性化体验受益者：获得基于记忆系统的个性化教学

### 2.3 管理员 (Admin)
- 系统管理者：管理用户账号、系统配置
- 插件管理者：启用/禁用系统插件、调整插件配置
- 运维监控者：监控系统健康状态、性能指标

## 3. 功能需求

### 3.1 用户系统 (V1.0)
- **多角色登录**: 同一账号支持老师/学生/管理员身份
- **身份切换**: 界面根据角色动态适配，展示不同功能模块
- **权限控制**: 功能按角色严格隔离，基于RBAC权限模型
- **JWT认证**: 基于JWT令牌的无状态认证，支持令牌刷新
- **会话管理**: 支持多设备登录、会话超时自动注销

### 3.2 知识库管理 (V1.0)
- **文档上传**: 支持PDF/DOCX/TXT/MD格式，单文件最大50MB
- **内容解析**: 自动提取文本内容，支持分块处理（chunk_size: 1000, overlap: 200）
- **向量化存储**: 文档内容自动向量化，存入Chroma DB向量数据库
- **语义检索**: 基于向量相似度的语义搜索，支持混合检索模式
- **分类标签**: 支持多级分类和标签，方便知识组织管理
- **批量操作**: 支持批量上传、删除、分类调整
- **数据隔离**: 每位教师的知识库完全独立，互不干扰

### 3.3 对话系统 (V1.0)
- **苏格拉底式教学**: 引导式提问，启发学生主动思考而非直接给出答案
- **上下文构建**: 自动整合知识库检索结果、学生记忆、对话历史构建完整上下文
- **多模型支持**: 支持OpenAI GPT-4/GPT-3.5-turbo等多种大模型，可通过配置切换
- **对话策略**: 可配置对话风格（温度、最大轮次、最大token数等）
- **实时交互**: 低延迟对话体验，支持流式输出
- **上下文保持**: 维持对话连贯性，最大上下文长度4000 tokens
- **记录保存**: 完整对话历史记录，支持按时间、关键词检索

### 3.4 记忆系统 (V1.0)
- **对话记忆**: 自动提取对话中的重要信息并存储
- **学习进度记忆**: 跟踪学生在各知识点上的学习进度
- **个性特征记忆**: 记录学生的学习偏好、特点、薄弱环节
- **记忆检索**: 在对话时自动检索相关记忆，增强个性化体验
- **记忆类型**: 支持conversation、learning_progress、personality_traits三种类型
- **存储管理**: 基于SQLite存储，每用户最多1000条记忆，TTL 30天

### 3.5 增强功能 (V2.0)
- **高级记忆**: 记忆强度衰减机制，模拟人类遗忘曲线
- **数据分析**: 学习情况统计图表，教师可视化看板
- **留言系统**: 学生可向教师数字分身留言，教师可查看和回复
- **导出功能**: 支持对话记录、学习报告导出

### 3.6 扩展功能 (V3.0)
- **多租户支持**: 支持多机构/学校独立部署
- **插件市场**: 第三方插件安装和管理
- **API开放**: 第三方系统集成接口
- **多端适配**: Web/APP/小程序全端支持
- **智能推荐**: 基于学习数据的个性化内容推荐

## 4. 架构技术需求

### 4.1 Harness插件化架构
- **插件接口**: 统一的Plugin接口定义（Name/Version/Type/Init/Execute/HealthCheck/Destroy）
- **插件类型**: 支持knowledge、dialogue、memory、auth、analytics、export六种插件类型
- **插件管理**: 支持插件的注册、注销、热插拔、配置热重载
- **插件隔离**: 插件间通过事件总线通信，避免直接依赖

### 4.2 管道系统
- **管道定义**: 通过YAML配置定义插件执行顺序和流程
- **预定义管道**:
  - `student_chat`: 学生对话流程（认证 → 记忆 → 知识检索 → 对话生成）
  - `teacher_management`: 教师管理流程（认证 → 知识库操作）
- **执行控制**: 支持超时控制（默认30s）、错误处理、结果聚合
- **管道监控**: 执行轨迹记录、性能指标收集

### 4.3 配置驱动
- **主配置文件**: `configs/harness.yaml` 管理系统全局配置
- **环境配置**: 支持development/test/production多环境配置覆盖
- **插件配置**: 每个插件独立配置，支持运行时更新
- **敏感信息**: 通过环境变量注入（如 `${OPENAI_API_KEY}`、`${JWT_SECRET}`）

### 4.4 事件系统
- **事件总线**: 内存事件总线，支持发布/订阅模式
- **事件类型**: 插件生命周期事件、健康检查事件、业务事件
- **异步处理**: 支持事件的异步处理，不阻塞主流程

## 5. 非功能需求

### 5.1 性能要求
- API响应时间: < 200ms（不含大模型调用）
- 对话响应时间: < 2秒（含大模型调用）
- 文档处理速度: < 30秒/文档
- 系统可用性: 99.9%
- 并发支持: > 1000用户
- 插件执行超时: 30s

### 5.2 安全要求
- 数据加密: HTTPS传输加密 + AES-256-GCM存储加密
- 权限控制: 基于JWT的RBAC权限模型
- 审计日志: 完整操作日志记录，保留90天
- API限流: 100请求/分钟，突发20请求
- CORS控制: 严格的跨域访问策略

### 5.3 兼容性
- 前端框架: Taro跨端小程序框架 + Vant Weapp组件库
- 浏览器: Chrome/Safari/Firefox
- 移动端: iOS/Android（通过小程序）
- 文件格式: PDF/DOCX/TXT/MD

### 5.4 可观测性
- 健康检查: 每30秒自动检查系统和插件健康状态
- 指标监控: Prometheus格式指标暴露（端口9090）
- 结构化日志: JSON格式日志输出，支持日志级别配置

## 6. 数据需求

### 6.1 数据模型
```
teachers (教师) → knowledge_documents (知识文档)
students (学生) → conversation_history (对话历史)
students (学生) → student_memories (学生记忆)
users (用户) → roles (角色) → permissions (权限)
```

### 6.2 存储方案
- **关系型数据**: SQLite（V1.0），可扩展至MySQL
- **向量数据**: Chroma DB，支持语义检索
- **文件存储**: 本地文件系统（V1.0），支持云存储扩展

### 6.3 数据流
1. 文档上传 → 内容解析 → 分块处理 → 向量化 → 存入Chroma DB
2. 用户提问 → 记忆检索 → 知识库语义检索 → 构建上下文 → 大模型生成回复
3. 对话完成 → 记忆提取 → 记忆存储/更新 → 学习进度更新

## 7. 接口需求

### 7.1 RESTful API
```
POST   /api/auth/login              # 用户登录
POST   /api/auth/refresh            # 刷新令牌

GET    /api/teachers/{id}/documents  # 获取教师文档列表
POST   /api/teachers/{id}/documents  # 上传文档
DELETE /api/teachers/{id}/documents/{docId}  # 删除文档

POST   /api/students/{id}/chat       # 发送对话消息
GET    /api/students/{id}/history     # 获取对话历史
GET    /api/students/{id}/memories    # 获取学生记忆

GET    /api/system/health            # 系统健康检查
GET    /api/system/plugins           # 插件列表
GET    /api/system/metrics           # 系统指标
```

### 7.2 大模型接口
- 支持OpenAI API格式（兼容多种大模型服务）
- 支持文本生成（GenerateResponse）和文本向量化（EmbedText）
