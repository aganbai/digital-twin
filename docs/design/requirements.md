# 需求规格说明书

## 1. 项目背景

为教育行业提供AI数字分身解决方案，让教师能够创建自己的智能助教，为学生提供7x24小时的个性化教学服务。

### 1.1 项目概述

- **项目名称**: 数字分身系统 (Digital Twin System)
- **技术栈**: Go + Gin + Taro + Chroma DB + 大模型API
- **架构模式**: Harness插件化架构（配置驱动、管道编排）
- **部署模式**: 单机部署（Docker Compose）

### 1.2 项目目标

- 实现多老师多知识库支持，每位教师拥有独立的数字分身
- 支持学生个性化记忆管理，提供因材施教的学习体验
- 采用Harness插件化架构，提供可插拔的AI能力扩展
- 配置驱动开发，降低运维复杂度，支持快速迭代
- 建立师生授权体系，支持教师个性化管理学生

## 2. 用户角色

### 2.1 教师 (Producer)
- 知识内容生产者：上传和管理教学文档、知识库（文本/文件/URL）
- 学生管理者：邀请学生、审批申请、查看学习进度、撰写评语
- 教学数据分析者：分析学生学习情况、优化教学策略
- 数字分身定制者：配置分身的对话风格、针对每个学生设置个性化问答风格
- 作业评审者：查看学生作业、撰写点评

### 2.2 学生 (Consumer)
- 知识消费者：与数字分身进行苏格拉底式对话学习
- 学习进度被跟踪者：学习行为和进度被系统记录
- 个性化体验受益者：获得基于记忆系统的个性化教学
- 作业提交者：上传作业或成果，接受AI和教师点评
- 授权申请者：申请使用教师的数字分身

### 2.3 管理员 (Admin)
- 系统管理者：管理用户账号、系统配置
- 插件管理者：启用/禁用系统插件、调整插件配置
- 运维监控者：监控系统健康状态、性能指标

## 3. 功能需求

### 3.1 用户系统 (V1.0 ✅)
- **微信登录**: 微信小程序一键登录，code 换取 openid
- **角色选择**: 新用户登录后选择教师/学生角色
- **JWT认证**: 基于JWT令牌的无状态认证，支持令牌刷新（7天宽限期）
- **权限控制**: 功能按角色严格隔离

### 3.2 知识库管理 (V1.0 ✅ 基础 / V2.0 增强)
- **文本录入**: 教师手动输入文档标题和内容 (V1.0 ✅)
- **文档上传**: 支持PDF/DOCX/TXT/MD格式，单文件最大50MB (V2.0)
- **URL导入**: 输入URL自动抓取网页内容入库 (V2.0)
- **向量化存储**: 文档内容自动向量化，存入Chroma DB向量数据库 (V1.0 ✅)
- **语义检索**: 基于向量相似度的语义搜索 (V1.0 ✅)
- **分类标签**: 支持多标签分类 (V1.0 ✅)
- **数据隔离**: 每位教师的知识库完全独立 (V1.0 ✅)

### 3.3 对话系统 (V1.0 ✅ 基础 / V2.0 增强)
- **苏格拉底式教学**: 引导式提问，启发学生主动思考 (V1.0 ✅)
- **上下文构建**: 自动整合知识库检索结果、学生记忆、对话历史 (V1.0 ✅)
- **管道编排**: student_chat 管道按配置顺序执行4个插件 (V1.0 ✅)
- **多模型支持**: 支持通义千问/DeepSeek等OpenAI兼容格式模型 (V1.0 ✅)
- **流式输出**: SSE流式推送，前端逐字渲染 (V2.0)
- **师生授权鉴权**: 未授权学生无法对话 (V2.0)
- **个性化问答风格**: 教师可针对每个学生设置不同的对话风格 (V2.0)
- **记录保存**: 完整对话历史记录 (V1.0 ✅)

### 3.4 记忆系统 (V1.0 ✅ 基础 / V2.0 增强)
- **对话记忆**: 对话后自动提取关键信息并存储 (V1.0 ✅)
- **学习进度记忆**: 跟踪学生在各知识点上的学习进度 (V1.0 ✅)
- **个性特征记忆**: 记录学生的学习偏好、特点、薄弱环节 (V1.0 ✅)
- **记忆检索**: 在对话时自动检索相关记忆 (V1.0 ✅)
- **记忆衰减**: 基于遗忘曲线的记忆强度衰减机制 (V2.0)

### 3.5 师生关系管理 (V2.0 🆕)
- **教师注册增强**: 必填学校和分身描述，教师名称+学校唯一
- **师生授权**: 教师可邀请学生，学生可申请使用分身
- **教师评语**: 教师可对学生撰写评语，含学习进度摘要
- **个性化问答风格**: 教师可针对每个学生设置对话风格参数

### 3.6 作业系统 (V2.0 🆕)
- **作业提交**: 学生可上传作业或成果（文本+文件）
- **AI点评**: 基于教师知识库自动生成AI点评
- **教师点评**: 教师可手动撰写点评和评分
- **作业管理**: 作业列表、详情查看、状态跟踪

### 3.7 数据分析 (V2.0)
- **教师看板**: 学生对话统计、知识库热度、学习进度概览
- **对话导出**: 对话记录导出为TXT

## 4. 架构技术需求

### 4.1 Harness插件化架构 (V1.0 ✅)
- **插件接口**: 统一的Plugin接口定义（Name/Version/Type/Init/Execute/HealthCheck/Destroy）
- **插件类型**: 支持knowledge、dialogue、memory、auth四种核心插件
- **插件管理**: 支持插件的注册、注销、配置驱动
- **插件隔离**: 插件间通过管道数据流转通信

### 4.2 管道系统 (V1.0 ✅)
- **管道定义**: 通过YAML配置定义插件执行顺序和流程
- **预定义管道**: student_chat（认证→记忆→知识检索→对话生成）
- **执行控制**: 超时控制（默认30s）、错误处理、结果聚合
- **管道监控**: 执行轨迹记录、耗时统计

### 4.3 配置驱动 (V1.0 ✅)
- **主配置文件**: `configs/harness.yaml` 管理系统全局配置
- **环境变量替换**: 支持 `${VAR_NAME}` 语法
- **配置校验**: 必填字段缺失时返回明确错误
- **敏感信息**: 通过环境变量注入

## 5. 非功能需求

### 5.1 性能要求
- API响应时间: < 200ms（不含大模型调用）
- 对话响应时间: < 2秒（含大模型调用）
- 文档处理速度: < 30秒/文档
- 插件执行超时: 30s

### 5.2 安全要求
- 数据传输: HTTPS（V2.0 生产环境）
- 权限控制: 基于JWT的角色权限模型
- API限流: 100请求/分钟（V2.0 落地）
- CORS控制: 严格的跨域访问策略
- 师生授权: 未授权学生无法访问教师分身

### 5.3 兼容性
- 前端框架: Taro 3.x 跨端小程序框架
- 移动端: 微信小程序
- 文件格式: PDF/DOCX/TXT/MD

### 5.4 可观测性
- 健康检查: 系统和插件健康状态（含 uptime、database 状态）
- 结构化日志: JSON格式日志输出
- 监控告警: 健康检查失败告警（V2.0）

## 6. 数据需求

### 6.1 数据模型

```
users (用户) → documents (知识文档)
users (用户) → conversations (对话历史)
users (用户) → memories (学生记忆)
teacher_student_relations (师生关系)
teacher_comments (教师评语)
student_dialogue_styles (问答风格配置)
assignments (学生作业) → assignment_reviews (作业点评)
```

### 6.2 存储方案
- **关系型数据**: SQLite（WAL 模式，V2.0 生产加固）
- **向量数据**: Chroma DB（V2.0 Docker 持久化）
- **文件存储**: 本地文件系统（uploads/ 目录）
