## 三、失败用例详情 (P0/P1)

### ❌ B-05 / AD-02: 教师禁止独立创建分身 【P0 阻塞】

**失败层级**: API
**失败步骤**: 第1步 — POST /api/personas with role=teacher
**预期结果**: 返回 400，code=40040
**实际结果**: 返回 200，code=0，成功创建分身
**HTTP 状态码**: 200
**错误信息**: message="success"
**相关代码**: `src/backend/api/persona_handler.go` 或对应路由处理函数

**问题归属**: 后端 (dev_backend_agent)
**修复方向**: 
1. 在 `POST /api/personas` handler 中增加 role==teacher 的前置校验
2. 当请求体 role 为 teacher 时直接返回 40040 错误
3. 确保 middleware 或 handler 层拦截教师角色的独立创建请求

---

### ❌ AD-04: 已删除接口返回404 【P0 阻塞】

**失败层级**: API
**失败步骤**: 第1步 — PUT /api/personas/:id/{switch,activate,deactivate}
**预期结果**: 三个接口均返回 404
**实际结果**: 三个接口均返回 200 success
**HTTP 状态码**: 200
**相关代码**: router.go 路由注册

**问题归属**: 后端 (dev_backend_agent)
**修复方向**:
1. 检查 router.go 中是否仍注册了这三个路由
2. 如已注释/删除路由但编译未生效，重新 `go build`
3. 如路由仍在，从路由表彻底移除

---

### ❌ B-02 / AD-03: 分身列表缺少班级绑定字段 【P1 重要】

**失败层级**: API
**失败步骤**: GET /api/personas 返回值检查
**预期结果**: 每个 teacher persona 含 bound_class_id / bound_class_name / is_public
**实际结果**: 仅返回基础字段(id/nickname/school/description)，无班级绑定信息
**相关代码**: `GET /api/personas` handler 的 SQL 查询和响应构造

**问题归属**: 后端 (dev_backend_agent)
**修改方向**:
1. SQL 查询增加 LEFT JOIN classes 表
2. 响应结构增加 bound_class_id/bound_class_name/is_public 字段
3. 仅对 teacher 类型且有绑定的分身填充这些字段

---

### ❌ S-07: 管理员用户列表查询失败 【P0 阻塞】

**失败层级**: API
**失败步骤**: GET /api/admin/users
**预期结果**: 返回用户分页列表
**实际结果**: error="sql: Scan error on column index 5, name 'email': converting NULL to string is unsupported"
**HTTP 状态码**: 200 (但 success=false)
**问题归属**: 后端 (dev_backend_agent)
**修改方向**:
1. users 表中 email 字段允许 NULL
2. Scan 时使用 sql.NullString 替代 string
3. 或在查询时使用 COALESCE(email,'') 处理 NULL
