## 数据管理脚本

### 使用方法

```bash
# 创建测试数据
./scripts/manage_test_data.sh create

# 验证测试数据
./scripts/manage_test_data.sh verify

# 清理测试数据
./scripts/manage_test_data.sh clean

# 重置测试数据（清理后重新创建）
./scripts/manage_test_data.sh reset
```

### 脚本功能说明

#### create - 创建测试数据
执行SQL脚本插入以下数据：
1. 创建教师和学生账号
2. 为账号创建默认分身
3. 创建测试班级
4. 建立师生授权关系
5. 将学生加入班级

#### verify - 验证测试数据
检查以下数据是否完整：
- 教师账号存在
- 学生账号存在
- 教师分身存在
- 学生分身存在
- 测试班级存在
- 师生关系已建立
- 学生已加入班级

#### clean - 清理测试数据
删除所有测试相关数据：
- 删除班级成员记录
- 删除班级记录
- 删除师生关系记录
- 删除分身记录
- 删除用户账号

#### reset - 重置测试数据
依次执行：clean → create → verify

## 数据库表结构说明

### 核心表

#### users - 用户表
存储教师和学生账号信息。

#### personas - 分身表
存储用户的分身信息，每个用户可以有多个分身。

#### classes - 班级表
存储班级信息，班级关联教师分身。

#### class_members - 班级成员表
存储班级的学生成员关系。

#### teacher_student_relations - 师生关系表
存储教师和学生的授权关系。

## 数据插入SQL脚本

SQL脚本位置：`scripts/insert_test_data.sql`

脚本包含以下步骤：
1. 清理可能存在的旧测试数据
2. 创建教师和学生账号
3. 创建分身记录
4. 更新用户的默认分身ID
5. 创建测试班级
6. 建立师生授权关系
7. 将学生加入班级
8. 验证数据创建结果

## 注意事项

1. **密码加密**：测试账号密码使用bcrypt加密存储，明文密码为 `test123456`
2. **数据隔离**：测试数据使用特定的班级名称（冒烟测试班-自动化），避免与生产数据冲突
3. **幂等性**：SQL脚本支持重复执行，会自动清理旧数据
4. **数据完整性**：脚本会创建完整的数据关系链，确保测试用例可以正常运行

## 维护指南

### 更新测试数据
如果需要修改测试数据结构：
1. 编辑 `scripts/insert_test_data.sql`
2. 运行 `./scripts/manage_test_data.sh reset`
3. 验证数据完整性

### 问题排查
如果测试数据出现问题：
1. 运行 `./scripts/manage_test_data.sh verify` 检查数据完整性
2. 如果验证失败，运行 `./scripts/manage_test_data.sh reset` 重置数据
3. 检查数据库日志和错误信息
