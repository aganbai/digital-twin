# V2.0 迭代13 需求文档

## 概述

本迭代主要包含两大方向：

1. **UI优化调整** - 统一各端UI体验，优化交互细节
2. **教材配置流程重构** - 将教材配置整合到班级创建/编辑流程中，移除独立入口

## 二、功能需求

### R1. 教材配置整合到班级创建

#### 1.1 功能描述

在创建班级页面增加可选的教材配置区域，用户可以在创建班级时同时配置教材信息（学段、年级、学科、教材版本、教学进度）。该配置为非必填项。

#### 1.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 配置位置 | 班级创建表单中新增"教材配置"区域（位于班级信息之后） |
| 配置字段 | 学段（必填）、年级（根据学段联动）、学科（可多选）、教材版本（可多选）、教学进度（可选） |
| 默认状态 | 教材配置区域默认折叠，点击展开后显示 |
| 必填性 | 整个教材配置为可选，但一旦展开填写，学段为必填 |
| 数据绑定 | 教材配置绑定到班级关联的teacher_persona_id |

#### 1.3 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| curriculum_config.grade_level | string | 条件 | 学段：primary（小学）、middle（初中）、high（高中）、university（大学）、adult_life（成人生活）、adult_professional（成人职业） |
| curriculum_config.grade | string | 条件 | 年级，根据学段动态显示选项 |
| curriculum_config.subjects | string[] | 条件 | 学科数组，可多选 |
| curriculum_config.textbook_versions | string[] | 否 | 教材版本数组 |
| curriculum_config.custom_textbooks | string[] | 否 | 自定义教材名称（大学及以上学段使用） |
| curriculum_config.current_progress | string | 否 | 当前教学进度描述 |

#### 1.4 学段与年级对应关系

| 学段 | 年级选项 |
|------|----------|
| primary | 一年级、二年级、三年级、四年级、五年级、六年级 |
| middle | 七年级、八年级、九年级 |
| high | 高一、高二、高三 |
| university | 无年级选项（大学及以上手动填写教材） |
| adult_life | 无年级选项 |
| adult_professional | 无年级选项 |

#### 1.5 交互流程

```
用户进入班级创建页面
    ↓
填写班级基本信息（名称、描述、公开设置）
    ↓
点击"教材配置（可选）"展开区域
    ↓
选择学段 → 联动显示对应年级选项
    ↓
选择年级（如有）
    ↓
选择教材版本（可多选，大学及以上显示手动输入）
    ↓
选择教学学科（可多选）
    ↓
填写当前教学进度（可选）
    ↓
点击创建 → 后端同时创建班级和教材配置
```

#### 1.6 验收标准

- [ ] 班级创建页面显示"教材配置（可选）"折叠区域
- [ ] 展开后显示完整的教材配置表单
- [ ] 学段选择后联动更新年级选项
- [ ] 大学及以上学段支持手动输入教材名称
- [ ] 提交后班级和教材配置同时创建成功
- [ ] 如果不展开教材配置区域，创建班级不带教材配置

### R2. 编辑班级时支持修改教材配置

#### 2.1 功能描述

在编辑班级页面支持查看和修改已绑定的教材配置，包括新增、修改、删除教材配置。

#### 2.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 入口位置 | 班级编辑页面新增"教材配置"区域 |
| 显示逻辑 | 如有已有配置，展开显示；如无，折叠显示（同创建页） |
| 编辑能力 | 支持修改所有教材配置字段 |
| 删除能力 | 支持删除已有教材配置 |
| 新增能力 | 无配置时支持新增配置 |

#### 2.3 交互流程

```
用户进入班级编辑页面
    ↓
页面加载时获取班级详情，包括关联的教材配置
    ↓
如有教材配置，自动展开显示
    ↓
用户可修改任意配置字段
    ↓
点击保存 → 同时更新班级信息和教材配置
```

#### 2.4 验收标准

- [ ] 编辑页面加载时获取并显示已有教材配置
- [ ] 支持修改所有教材配置字段
- [ ] 支持删除已有教材配置
- [ ] 无配置时支持新增配置
- [ ] 保存后班级和教材配置同步更新

### R3. 废弃原有教材配置独立入口

#### 3.1 功能描述

废弃原独立的教材配置页面（curriculum-config），在小程序端个人中心页面移除"教材配置"菜单入口。

#### 3.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 移除页面 | pages/curriculum-config/index.tsx 保留但不再从个人中心进入 |
| 移除入口 | profile/index.tsx 中的"教材配置"菜单项 |
| 页面保留 | curriculum-config页面代码保留，但标记为废弃 |
| 数据保留 | 已有教材配置数据不受影响 |

#### 3.3 验收标准

- [ ] 个人中心页面不再显示"教材配置"菜单
- [ ] curriculum-config页面代码保留但不再主动维护
- [ ] 已有教材配置数据正常可用

### R4. 后端API扩展

#### 4.1 创建班级API扩展

**POST /api/classes**

请求体扩展：
```json
{
  "name": "班级名称",
  "description": "班级描述",
  "is_public": true,
  "curriculum_config": {
    "grade_level": "primary",
    "grade": "三年级",
    "subjects": ["数学", "语文"],
    "textbook_versions": ["人教版", "北师大版"],
    "custom_textbooks": [],
    "current_progress": "第三单元 两位数乘法"
  }
}
```

#### 4.2 编辑班级API扩展

**PUT /api/classes/:id**

支持更新 curriculum_config 字段，逻辑与创建相同。

#### 4.3 班级详情API扩展

**GET /api/classes/:id**

响应体扩展：
```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "班级名称",
    "description": "班级描述",
    "is_public": true,
    "curriculum_config": {
      "id": 1,
      "grade_level": "primary",
      "grade": "三年级",
      "subjects": ["数学", "语文"],
      "textbook_versions": ["人教版"],
      "current_progress": "第三单元"
    }
  }
}
```

## 三、UI/UX设计说明

### 3.1 整体交互风格

- **折叠面板**：教材配置区域使用可折叠面板，默认收起
- **联动选择**：学段选择后动态更新年级选项
- **标签多选**：学科和教材版本使用标签式多选
- **表单验证**：学段为教材配置中唯一必填项

### 3.2 组件设计

#### 折叠面板组件

| 属性 | 说明 |
|------|------|
| 标题 | "📚 教材配置（可选）" |
| 默认状态 | 折叠 |
| 展开图标 | 右侧箭头，展开时旋转180度 |

#### 级联选择器

| 属性 | 说明 |
|------|------|
| 学段 | Picker选择器 |
| 年级 | 根据学段动态选项的Picker |

#### 标签选择器

| 属性 | 说明 |
|------|------|
| 学科 | 多选标签，支持取消选择 |
| 教材版本 | 多选标签，支持取消选择 |

#### 自定义输入

| 属性 | 说明 |
|------|------|
| 触发条件 | 大学及以上学段 |
| 输入方式 | Input + 添加按钮 |
| 显示方式 | 标签列表，可删除 |

### 3.3 色彩规范

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| 折叠面板标题 | #1890FF | 主题蓝色 |
| 选中标签背景 | #1890FF | 主题蓝色 |
| 未选标签背景 | #F5F5F5 | 浅灰色 |
| 必填标记 | #FF4D4F | 红色 |

## 六、技术难点与风险点

| 风险点 | 风险等级 | 应对措施 |
|--------|---------|---------|
| 学段年级联动逻辑 | 低 | 前端维护映射关系，避免后端复杂联动 |
| 数据模型兼容性 | 中 | 保持persona_id绑定关系不变 |
| H5端表单复杂度 | 中 | 使用Element Plus的Collapse和Select组件 |
| 存量数据迁移 | 低 | 无需迁移，已有配置通过编辑班级可访问 |

## 七、UI变更说明

本迭代涉及UI变更，已生成UI规范文档：`ui_spec.md`

**主要UI变更**：
1. 小程序端班级创建页新增教材配置折叠区域
2. 小程序端班级编辑页新增教材配置折叠区域
3. 小程序端个人中心页移除"教材配置"菜单
4. H5教师端Classes.vue创建/编辑弹窗新增教材配置表单

**TabBar & 对话页 UI 修正**（2026-04-09 补充）：

| 修正项 | 修正前 | 修正后 | 原因 |
|--------|--------|--------|------|
| 小程序学生端 TabBar | 聊天/对话/发现/我的 (4项) | 聊天/发现/我的 (3项) | "对话"由聊天列表内点击进入，无需独立 Tab |
| 小程序教师端 TabBar | 聊天/对话/发现/学生/知识库/我的 (6项) | 聊天/学生/知识库/我的 (4项) | 教师端不需要"对话"和"发现"两个 Tab |
| 对话页(学生端) 导航栏右上角 | "详情"文字按钮 | 移除 | 与小程序原生胶囊按钮区域冲突 |
| 对话页语音按钮样式 | 🎤 emoji | 白色喇叭图标 🔊（圆形背景） | 更贴近真实小程序UI风格 |
| 对话页 + 号面板 | 仅含附件/拍照等 | 新增"新会话"入口 | 学生端支持向同一老师新建对话 |
| 对话页(教师端) | 与学生端相同样式 | 区分展示：导航栏显示学生昵称；右侧有接管/退出接管按钮；无新会话按钮 | 教师端从聊天列表查看学生对话，角色视角不同 |
