# V2.0 迭代12 需求文档

## 概述

本迭代旨在增强老师分身的会话管理能力。核心功能包括：
1. **用户中断能力** - 在SSE流式回复时点击中断按钮停止AI回复
2. **会话列表入口** - 聊天页左上角增加会话列表入口，展示当前分身下所有会话
3. **新会话按钮移动** - 将新增会话按钮移到会话列表顶部
4. **指令系统** - 支持#开头的指令如#新会话、#给老师留言

## 二、功能需求

### R1. 用户中断能力

#### 1.1 功能描述

在AI流式回复过程中，用户可以点击"停止生成"按钮中断AI的回复。中断后，已接收的内容应保留在聊天记录中，并停止继续接收新内容。

#### 1.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 触发时机 | SSE流式回复进行中（`isStreaming = true`） |
| 按钮位置 | 替换原有的"发送"按钮位置（输入框右侧） |
| 按钮样式 | 红色或橙色，突出显示"停止"或"✕" |
| 中断行为 | 1. 调用SSE流的abort()方法中断连接<br>2. 停止接收新内容<br>3. 将已接收的streamingContent转为完整消息<br>4. 恢复正常输入状态 |
| 数据保留 | 中断前已接收的内容保留在聊天记录中 |
| 继续对话 | 中断后用户可继续发送消息，正常对话流程不变 |

#### 1.3 交互流程

```
用户发送消息
    ↓
AI开始流式回复 (isStreaming = true)
    ↓
显示"停止生成"按钮 (替换发送按钮)
    ↓
用户点击"停止生成"按钮
    ↓
中断SSE连接 (streamTaskRef.current.abort())
    ↓
将已接收内容转为消息
    ↓
恢复正常输入状态 (isStreaming = false)
    ↓
显示正常发送按钮
```

#### 1.4 验收标准

- [ ] SSE流式回复进行中时，显示"停止生成"按钮
- [ ] 点击"停止生成"按钮后，AI立即停止回复
- [ ] 已接收的AI回复内容保留在聊天记录中
- [ ] 中断后输入框恢复正常，可继续发送新消息
- [ ] 中断后思考过程面板隐藏
- [ ] 中断后流式状态正确重置（isStreaming = false, setStreamingContent('')）

### R2. 会话列表入口

#### 2.1 功能描述

在聊天页左上角增加会话列表入口，点击后展开侧边栏，展示当前分身（老师）下的所有历史会话。

#### 2.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 入口位置 | 聊天页左上角导航栏，替换或新增在返回按钮左侧 |
| 入口样式 | 汉堡菜单图标（三横线）或会话列表图标 |
| 交互方式 | 遮罩层 + 侧边栏滑出（从左侧滑入） |
| 关闭方式 | 1. 点击遮罩层<br>2. 点击侧边栏内的关闭按钮 |
| 侧边栏宽度 | 屏幕宽度的 75% ~ 80% |
| 遮罩层 | 半透明黑色（rgba(0,0,0,0.5)） |

#### 2.3 会话列表内容

| 字段 | 说明 |
|------|------|
| 当前会话标识 | 显示"当前会话"标签 |
| 会话标题 | LLM自动生成的会话标题（session_titles表） |
| 最后一条消息 | 截断显示，最多30字 |
| 消息时间 | 格式化时间显示 |
| 消息数量 | 该会话的总消息数 |
| 当前标记 | 高亮显示正在进行的会话 |

#### 2.4 交互流程

```
用户点击左上角会话入口图标
    ↓
显示遮罩层
    ↓
侧边栏从左侧滑入
    ↓
展示当前分身下的所有会话列表
    ↓
用户点击某个历史会话
    ↓
切换到该会话，加载历史消息
    ↓
关闭侧边栏
```

#### 2.5 验收标准

- [ ] 聊天页左上角显示会话列表入口图标
- [ ] 点击入口图标后，侧边栏从左侧平滑滑入
- [ ] 侧边栏显示该分身下的所有历史会话
- [ ] 当前会话有"当前会话"标识
- [ ] 点击历史会话后，切换到该会话并更新消息列表
- [ ] 点击遮罩层或关闭按钮，侧边栏平滑收起
- [ ] 侧边栏打开时，背景聊天区域可滚动但不可交互

#### 3.2 需求详情

| 需求项 | 说明 |
|--------|------|
| 按钮位置 | 侧边栏（会话列表）顶部 |
| 按钮样式 | "+新会话"或"+ 新对话" |

```

    ↓

```

#### 3.4 验收标准

- [ ] 侧边栏顶部显示"+新会话"按钮

### R4. 指令系统

#### 4.1 功能描述

在输入框中支持以`#`开头的指令，用户可以通过输入特定指令触发系统级操作。

#### 4.2 支持的指令

| 指令 | 触发条件 | 功能 | 说明 |
|------|---------|------|------|
| `#新会话` | 输入框内容为`#新会话` | 创建新会话 | 重置消息列表，清空session_id |
| `#新对话` | 输入框内容为`#新对话` | 创建新会话 | 同#新会话 |
| `#给老师留言` | 输入框内容为`#给老师留言` | 留言模式 | 标记消息类型为teacher_message_direct |
| `#留言` | 输入框内容为`#留言` | 留言模式 | 同#给老师留言 |
| `#新话题` | 输入框内容为`#新话题` | 创建新会话 | 同#新会话（与迭代9保持一致） |

#### 4.3 指令识别逻辑

```javascript
// 指令正则表达式
const COMMAND_PATTERN = /^#(新会话|新对话|新话题|给老师留言|留言)$/

// 指令处理
const commandMatch = inputValue.trim().match(COMMAND_PATTERN)
if (commandMatch) {
  const command = commandMatch[1]
  switch(command) {
    case '新会话':
    case '新对话':
    case '新话题':
      // 创建新会话逻辑
      clearMessages()
      setSessionId('')
      Taro.showToast({ title: '已创建新会话', icon: 'success' })
      break
    case '给老师留言':
    case '留言':
      // 留言模式逻辑
      setInputValue('')
      setInputPlaceholder('请输入给老师的留言...')
      setIsMessageMode(true)
      break
  }
  return // 不作为普通消息发送
}
```

#### 4.4 指令反馈

| 指令类型 | 反馈方式 | 反馈内容 |
|---------|---------|---------|
| 新会话类 | Toast | "已创建新会话" |
| 留言类 | UI变化 | 输入框提示变化为"请输入给老师的留言..." |

#### 4.5 验收标准

- [ ] 输入`#新会话`并发送后，消息列表清空
- [ ] 输入`#新对话`并发送后，消息列表清空
- [ ] 输入`#新话题`并发送后，消息列表清空（支持迭代9指令）
- [ ] 输入`#给老师留言`并发送后，进入留言模式
- [ ] 指令不作为普通消息发送给AI
- [ ] 指令不区分中英文标点（仅识别半角#）
- [ ] 指令前后有空格也能正确识别
- [ ] 指令触发后输入框内容清空

## 三、API接口设计

### A1. 流式中断接口

#### GET /api/chat/stream/:session_id/abort

**描述**：中断指定会话的流式回复

**鉴权**：需要（Bearer Token）

**路径参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| session_id | string | ✅ | 会话ID |

**成功响应** `200`：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "aborted": true,
    "message": "流式回复已中断"
  }
}
```

**错误响应**：
| 场景 | HTTP | code | message |
|------|------|------|---------|
| 会话不存在 | 404 | 40004 | 会话不存在 |
| 无权操作 | 403 | 40001 | 无权限操作该会话 |

### A2. 获取分身会话列表（复用）

#### GET /api/sessions

**描述**：获取指定分身的会话列表（复用迭代9接口）

**鉴权**：需要（Bearer Token）

**查询参数**：
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| teacher_persona_id | number | ✅ | - | 教师分身ID |
| page | number | ❌ | 1 | 页码 |
| page_size | number | ❌ | 20 | 每页数量 |

**成功响应** `200`：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 10,
    "page": 1,
    "page_size": 20,
    "items": [
      {
        "session_id": "uuid-1",
        "title": "关于微积分的讨论",
        "last_message": "好的，那我们来详细讨论积分...",
        "message_count": 15,
        "updated_at": "2026-04-07T10:30:00Z"
      }
    ]
  }
}
```

### A3. 创建新会话（复用）

#### POST /api/sessions

**描述**：创建新会话（复用迭代9接口）

**鉴权**：需要（Bearer Token）

**请求体**：
```json
{
  "teacher_persona_id": 1,
  "initial_message": ""
}
```

**成功响应** `200`：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "session_id": "uuid-new"
  }
}
```

## 七、技术难点与风险点

| 风险点 | 风险等级 | 应对措施 |
|--------|---------|---------|
| SSE流式中断 | 中 | 使用AbortController API，确保连接正确关闭 |
| 会话切换状态同步 | 中 | 使用Zustand store统一管理会话状态 |
| 指令识别与普通消息区分 | 低 | 前端发送前预检查，指令不走发送流程 |
| 侧边栏动画性能 | 低 | 使用CSS transform，避免触发重排 |
