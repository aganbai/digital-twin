# V2.0 迭代5 需求规格说明书

## 1. 迭代概述

| 项目 | 说明 |
|------|------|
| **迭代名称** | V2.0 Sprint 5 - LlamaIndex 语义检索 + UX 重构 |
| **迭代目标** | 引入 LlamaIndex 语义检索替换内存向量存储，重构教师/学生首页 UX，合并管理页面 |
| **迭代周期** | ~3 周 |
| **交付标准** | 所有新功能通过集成测试 + 端到端冒烟验证 |
| **前置依赖** | V2.0 迭代4 全部完成（134 个集成测试通过） |

## 2. 迭代目标

### 2.1 核心目标

> **LlamaIndex 语义检索 + 教师 Dashboard 重构 + 学生首页优化 + 管理页面合并 + 对话附件支持**

具体来说：
1. **LlamaIndex 语义检索服务（R1）**：新增 Python LlamaIndex 服务，替换 Go 侧 `InMemoryVectorStore`，实现 Embedding 向量化 + 语义检索；Go 侧分块逻辑保留不动，Python 服务只负责向量存储和检索
2. **教师首页改为 Dashboard（R2）**：教师落地页从知识库页改为 Dashboard（数据统计 + 待审批 + 快捷操作），复用已有 `GET /api/personas/:id/dashboard` 接口
3. **学生1个老师直接进对话（R3）**：学生只有1个已授权老师时，跳过首页直接进入对话页
4. **师生管理 + 班级管理合并为"学生管理"（R4）**：合二为一，Tab 切换（全部学生 / 按班级 / 待审批 / 班级设置）
5. **评语改为"学生备注"（R5）**：评语仅作为 AI 个性化依据，学生端去掉评语入口，学生不可见
6. **分享码管理移入分身管理（R6）**：分享码从首页独立入口移入分身管理页
7. **作业支持在聊天窗口提交（R7）**：对话支持附件发送，AI 自动识别并点评（P1 优先级）
8. **广场独立化（R8）**：广场从首页内嵌拆为独立页面，从首页点击跳转（不新增 TabBar）

### 2.2 不在本迭代范围
- ❌ Docker 容器化部署（Docker Compose 编排放到后续迭代）
- ❌ HTTPS / Nginx 配置
- ❌ 安全加固 / API 限流
- ❌ 记忆衰减机制
- ❌ 旧数据迁移（无需将 InMemoryVectorStore 数据迁移到 LlamaIndex）
- ❌ ChromaDB（使用 LlamaIndex SimpleVectorStore 本地文件持久化）

### 2.3 与迭代4的关系
本迭代在迭代4的基础上，核心升级是**后端架构**（LlamaIndex 语义检索替换内存向量存储）和**前端 UX 重构**（教师 Dashboard、学生首页优化、管理页面合并）。后端改动集中在知识库插件的向量存储层替换（~4 处调用点）+ 新增 Python 服务；前端改动较大（6 个页面重构/新增）。

## 4. 架构设计

### 4.1 整体架构（改造后）

```
┌─────────────────────────────────────────────────────────────┐
│                    单机部署架构                                │
│                                                               │
│  ┌─────────────────────────────────────────────────────┐     │
│  │              Go 后端 :8080                            │     │
│  │                                                       │     │
│  │  API Layer → KnowledgePlugin → TextChunker (保留)     │     │
│  │              ↓                   Scope过滤 (保留)      │     │
│  │              VectorClient ──HTTP──→ Python服务         │     │
│  │                                                       │     │
│  │  DialoguePlugin → 通义千问 LLM API                    │     │
│  │  MemoryPlugin → SQLite                                │     │
│  └─────────────────────────────────────────────────────┘     │
│                                                               │
│  ┌─────────────────────────────────────────────────────┐     │
│  │         Python LlamaIndex 服务 :8100                  │     │
│  │                                                       │     │
│  │  FastAPI → LlamaIndex VectorStoreIndex                │     │
│  │            ↓                                          │     │
│  │            DashScope Embedding (text-embedding-v3)     │     │
│  │            ↓                                          │     │
│  │            SimpleVectorStore (本地JSON文件持久化)       │     │
│  └─────────────────────────────────────────────────────┘     │
│                                                               │
│  微信小程序 ──HTTP──→ Go 后端 :8080                           │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 职责划分

| 组件 | 职责 | 不做什么 |
|------|------|---------|
| **Go 后端** | 业务逻辑、权限校验、文本分块、Scope 过滤、文档元数据管理、对话管理 | 不做 embedding、不做向量存储 |
| **Python LlamaIndex 服务** | 接收已分好的 chunks → embedding 向量化 → 向量存储 → 语义检索 | 不做业务逻辑、不做权限校验、不做分块 |
| **SimpleVectorStore** | 向量持久化（本地 JSON 文件） | 不需要 ChromaDB |
| **通义千问 API** | LLM 对话 + Embedding 向量化 | — |

### 4.3 Go 侧改动点（最小侵入）

只需修改 `knowledge_plugin.go` 中 4 处调用 `p.vectorStore.XXX` 的地方，改为 HTTP 调用 Python 服务：

| 改动点 | 原调用 | 改为 |
|--------|--------|------|
| `handleAdd()` | `p.vectorStore.AddDocuments(...)` | `HTTP POST /api/v1/vectors/documents` |
| `handleSearch()` | `p.vectorStore.Search(...)` | `HTTP POST /api/v1/vectors/search` |
| `handlePipeline()` | `p.vectorStore.Search(...)` | `HTTP POST /api/v1/vectors/search` |
| `handleDelete()` | `p.vectorStore.DeleteByDocumentID(...)` | `HTTP DELETE /api/v1/vectors/documents/{doc_id}` |

其他所有代码（`TextChunker`、`filterByScope`、文档管理、预览等）**完全不动**。

### 4.4 Python 服务目录结构

```
src/knowledge-service/
├── app/
│   ├── __init__.py
│   ├── main.py              # FastAPI 入口，3 个核心接口 + 1 个健康检查
│   ├── config.py             # 配置（DashScope API Key 等）
│   └── index_service.py      # LlamaIndex 核心逻辑（~150行）
├── data/                     # SimpleVectorStore 持久化目录
├── requirements.txt          # llama-index, fastapi, uvicorn, dashscope
└── Dockerfile
```

## 5. 数据库设计

### 5.1 无表结构变更

本迭代**不需要数据库表结构变更**。主要改动是：
- 向量存储从 Go 内存迁移到 Python LlamaIndex 服务（外部存储）
- 评语接口权限调整（代码层面，不涉及表结构）
- 对话附件通过现有 `conversations` 表的 `message` 字段传递（JSON 格式扩展）

### 5.2 对话消息格式扩展（R7 - 附件支持）

对话消息的 `message` 字段支持附件信息，格式约定：

```json
{
  "text": "老师，帮我看看这篇作文",
  "attachment": {
    "url": "/uploads/2026/03/essay.pdf",
    "type": "pdf",
    "name": "我的作文.pdf",
    "size": 102400
  }
}
```

> 当 `message` 为纯文本时，保持原有字符串格式不变（向后兼容）。

## 6. 功能需求详细描述

### 6.1 R1 - LlamaIndex 语义检索服务

#### 6.1.1 功能描述
新增独立的 Python LlamaIndex 服务（端口 8100），提供向量存储和语义检索能力。Go 后端通过 HTTP 调用该服务，替换现有的 `InMemoryVectorStore`。

#### 6.1.2 用户故事
- 作为学生，我希望提问时能获得更准确的知识库匹配结果，即使我的提问用词和教材不完全一致
- 作为教师，我希望上传的教材能被更智能地检索，而不仅仅是关键词匹配

#### 6.1.3 技术要求
1. Python 服务使用 FastAPI 框架，暴露 3 个核心接口 + 1 个健康检查
2. 使用 LlamaIndex 的 `VectorStoreIndex` + `SimpleVectorStore`（本地文件持久化）
3. Embedding 模型使用通义千问 `text-embedding-v3`（通过 DashScope API）
4. Go 侧新增 `VectorClient` 结构体，封装 HTTP 调用逻辑
5. Go 侧 `TextChunker` 保留不动，分块后将 chunks 传给 Python 服务
6. Python 服务不可用时，Go 侧降级为不使用向量检索（返回空结果）

#### 6.1.4 接口清单
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/vectors/documents` | POST | 存储文档向量（接收已分好的 chunks） |
| `/api/v1/vectors/search` | POST | 语义检索（返回 top-k 相似文档块） |
| `/api/v1/vectors/documents/{doc_id}` | DELETE | 删除文档向量 |
| `/api/v1/health` | GET | 健康检查 |

## 11. 关键场景操作路径对比

| 场景 | 迭代4（当前） | 迭代5（目标） | 变化 |
|------|-------------|-------------|------|
| 教师登录（1个分身） | → 知识库管理页 | → **首页 Dashboard** | 落地页更合理 |
| 学生登录（1个老师） | → 首页 → 点老师 → 对话 | → **直接对话页** | **-2步** |
| 学生登录（0个老师） | → 首页（广场内嵌） | → 首页（**引导加入**） | 引导更明确 |
| 教师管理学生 | 师生管理 + 班级管理（2入口） | **学生管理**（1入口） | 合二为一 |
| 教师写评语 | 独立评语管理页 | 学生详情 → **学生备注** | 学生不可见 |
| 教师管理分享码 | 首页独立入口 | 分身管理 → **分享码** | 归属更清晰 |
| 学生提交作业 | 独立作业页（5+步） | 对话中**发送附件**（2-3步） | 路径大幅缩短 |
| 学生找新老师 | 首页内嵌广场 | 首页点"发现" → **独立广场页** | 首页更干净 |
| 知识库检索质量 | TF-IDF 关键词匹配 | **LlamaIndex 语义检索** | 质的飞跃 |
