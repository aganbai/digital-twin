# V2.0 迭代7 需求文档

## 概述

**迭代目标**：引入教材版本配置体系，增强Prompt鲁棒性，引入自适应RAG机制，建立用户反馈渠道，支持知识库批量上传，增强系统安全性，保护学生隐私，支持批量添加学生和分享码加入，扩展成人课程类型，班级通知推送，记忆合并时提炼用户画像持久化，语音输入与表情支持。

**核心价值**：
- 通过教材版本配置，让LLM回答与学生实际学习内容一致
- 教师只需选择年级和教材版本，系统自动完成教学适配
- 支持K12全科 + 成人教育（生活技能/职业培训）
- 批量上传课件、批量添加学生、分享码加入，降低操作成本
- LLM自主判断是否需要外部搜索，支持前沿问题
- API限流防护 + Prompt Injection防御，保护系统稳定和学生隐私
- 教师可通过微信服务通知向班级/指定学生推送消息到聊天页
- 语音输入和表情支持，提升交互体验

**功能剔除**：
- 移除学生页"我的作业"功能（作业提交、作业列表）
- 移除教师页"作业管理"功能（作业列表、作业点评）
- 原作业相关页面、API、E2E测试代码一并移除

**需求演进**：
1. 原始方案：知识包导入 → 发现LLM已内置K12基础知识，无需重复灌入
2. 调整方案：转向"教材版本配置 + Prompt约束"，更轻量精准
3. 架构调整：取消独立KAT子项目，文档处理内置到`scripts/`目录

## 功能需求

### R1. 年级/教材配置体系（P0）

**目标**：教师通过简单的年级选择，系统自动完成教学适配，通过Prompt约束LLM回答范围。

**用户流程**：教师注册/创建课程 → 选择学段、年级、教材版本、学科 → 系统自动注入Prompt → 后续对话自动受教学背景约束

**核心功能**：
1. **引导式配置界面**：
   - 选择学段和年级（8档学段，详见下方学段表）
   - 选择教材版本（默认推荐当地主流版本；大学及以上留空手动填写；成人学段无需教材版本）
   - 选择教学学科（可多选，支持K12全科：语文/数学/英语/物理/化学/生物/历史/地理/音乐/美术/体育等；成人学段替换为"课程类别"）
   - 设置当前教学进度（可选）
   - 支持多本教材配置（含辅导教材），允许教师手动填写

2. **教材版本管理**：支持人教版/北师大版/苏教版/沪教版/部编版/外研版等，按地区推荐默认版本

3. **配置信息自动注入**：配置信息自动注入System Prompt的【教学背景】段落，实时生效

**学段划分（8档）**：

| 学段标识 | 名称 | 年级范围 | 核心特征 |
|---------|------|---------|----------|
| `preschool` | 学前班 | 幼儿园大班~学前 | 游戏化、极简语言、安全感优先 |
| `primary_lower` | 小学低年级 | 1-3年级 | 活泼有趣、大量比喻故事、多鼓励 |
| `primary_upper` | 小学高年级 | 4-6年级 | 培养自主思考、简单推理、适当拓展 |
| `junior` | 初中 | 7-9年级 | 逐步专业化、构建知识体系 |
| `senior` | 高中 | 10-12年级 | 规范学科语言、强调方法论、联系考试 |
| `university` | 大学及以上 | 大学/研究生/博士 | 学术化、批判性思维、跨学科探索 |
| `adult_life` | 成人生活技能 | 烹饪/健身/手工等 | 实用导向、轻松友好、注重操作步骤 |
| `adult_professional` | 成人职业培训 | 职业技能/考证等 | 专业实用、案例驱动、对标行业标准 |

> **注**：每个学段对应独立的Prompt基础模板，约束LLM的语言风格、回答深度和互动方式。模板详细内容见独立文件 `docs/prompts/grade_level_templates.md`（开发时创建）。

**grade → grade_level 自动映射**：系统根据具体年级自动推断学段，无需教师手动选择。

**Prompt 组装优先级**（从高到低）：
1. 安全规则（R7） ← 最高优先级，不可覆盖
2. 学段基础模板 ← 硬约束，决定语言风格和回答深度
3. 教学风格模板（已有） ← 软约束，决定教学方法
4. 个性化教学要求（已有） ← 教师自定义补充
5. 教学背景 ← 教材版本、学科、进度
6. 学生个人信息（R8） ← 用于个性化服务
7. 行为约束（R4） ← 知识库为空时的行为规则
8. 相关知识 + 学生记忆 ← 动态内容

**成人课程特殊处理**：
- 教材版本留空（成人课程一般没有标准教材）
- "学科选择"替换为"课程类别"（如烹饪→中餐/西餐/烘焙；健身→力量/有氧/瑜伽）
- 前端选择成人学段时自动隐藏"教材版本"选项

**技术实现**：
- 数据库表：`teacher_curriculum_configs`
  - 核心字段：`teacher_id`, `persona_id`, `grade_level`(学段), `grade`(具体年级), `textbook_versions`(JSON数组), `subjects`(JSON数组), `current_progress`, `region`
- API：`GET/POST /api/curriculum-configs`、`GET /api/curriculum-versions`
- Prompt修改：`systemPromptBaseTemplate`中增加【教学背景】段落

### R2. Adaptive RAG - LLM自主决策外部检索（P1）

**目标**：让LLM自主判断是否需要外部搜索，支持前沿研究和时事问题。

**核心功能**：
1. Function Calling/Tool Use框架
2. `web_search`工具定义和System Prompt引导
3. 搜索结果摘要和上下文注入
4. 成本控制和开关配置

**触发场景**：前沿研究问题、时事内容、学生明确要求搜索、知识库检索置信度低

### R3. 用户反馈机制（P0）

**目标**：建立用户反馈渠道，持续改进产品体验（反馈面向开发者查看）。

**核心功能**：
1. 反馈提交API（类型：功能建议/Bug报告/内容问题/其他）
2. 反馈管理后台（列表查看、状态更新）
3. 前端反馈入口（我的页面、对话快捷反馈）
4. 反馈自动关联上下文（用户、设备、页面信息）

### R4. Prompt鲁棒性增强（P0）

**目标**：增强System Prompt的鲁棒性，确保知识库为空时行为正确，教材版本约束生效。

**核心功能**：
1. 知识库为空时诚实回答"不知道"，不编造信息
2. 教材版本约束注入（约束LLM回答范围）
3. 不确定时主动询问学生
4. 检索结果置信度判断和标记
5. 学段Prompt模板注入：根据`grade_level`自动选择对应模板
6. 学段优先级：学段模板为硬约束，教学风格为软约束，冲突时学段优先

### R5. 知识库批量上传（P1）

**目标**：支持教师一次性上传多个文件（最多20个，总大小≤100MB），支持PDF/DOCX/TXT/MD格式。

**架构设计**：Go端接收文件 → 异步调用Python脚本处理 → 前端轮询任务状态

**Go端（R5.1）**：
- `POST /api/documents/batch-upload`：接收文件，落盘到`data/pending-imports/{task_id}/`，创建`batch_tasks`记录，返回202 + task_id
- `GET /api/batch-tasks/:task_id`：查询任务进度
- 异步goroutine调用`scripts/import_documents.py`

**Python端（R5.2~R5.4）**：
- 入口：`scripts/import_documents.py`，接收`--task-id`、`--input-dir`、`--db-path`、`--persona-id`、`--knowledge-base-id`参数
- 扫描目录 → 逐个解析分块 → 向量化 → 写入SQLite数据库
- 支持部分成功：单文件失败不影响其他文件
- 向量化失败重试3次

**错误处理**：单文件失败跳过继续；全部失败标记`failed`；部分失败标记`partial_success`

### R6. API限流/安全加固（P1）

**目标**：为生产环境增加安全防护，防止API滥用。

**核心功能**：
1. 全局请求限流中间件
2. 对话接口单独限流
3. 文件上传安全校验
4. 敏感操作日志审计

### R7. 学生隐私防护 - Prompt Injection防御（P0）

**目标**：保护学生隐私，防止通过prompt injection获取教师对学生的评价信息。

**敏感内容**：教师对学生的个人评语（存在于学生记忆中）。回答中严禁出现对学生本身的评价，只能针对事的评价（某个作业、某个问题、某个答案等）。

**防护层级**：
1. **System Prompt安全指令（P0）**：最高优先级安全规则，禁止透露学生记忆和system prompt内容，遇绕开指令礼貌拒绝
2. **记忆注入脱敏（P0）**：对`personality_traits`类记忆加敏感标记，用内部标签包裹
3. **输出过滤（P2，可选）**：后端正则检查LLM输出是否泄露敏感信息

**实现**：修改`src/plugins/dialogue/prompt.go`中的`systemPromptBaseTemplate`和`buildMemoryText`方法，约20行代码。

### R8. 批量添加学生（P0）

**目标**：降低教师添加学生的操作成本，支持LLM智能解析、分享码加入、引导丰富学生信息。

#### R8.1 文本粘贴LLM智能解析
- 教师粘贴文本（如花名册） → LLM解析提取学生信息 → 列表展示供教师确认/修改 → 批量创建
- API：`POST /api/students/parse-text`、`POST /api/students/batch-create`

#### R8.2 分享码加入（复用现有 `persona_shares` 表）
- 复用现有分享码机制（`persona_shares` 表 + `/api/shares` 路由），不新建 `share_codes` 表
- 教师生成分享码（6-8位字母数字），配置有效期（1/3/7/30天/永不过期）和人数上限（最多100人）
- 学生通过分享码自助加入课程
- 教师可随时废止分享码
- API：复用现有 `POST/GET/DELETE /api/shares`、`POST /api/shares/join`

#### R8.3 学生信息丰富化
- 界面提示引导教师多填写学生信息："💡 填写越多学生信息，AI个性化辅导越精准哦！"
- **必填**：姓名、性别
- **选填推荐**：年龄、学号、擅长/薄弱学科、学习风格偏好
- **选填个性化**：性格特点（多选标签）、兴趣爱好、特长、家长备注
- **扩展**：JSON自定义字段
- 丰富的学生信息自动注入对话Prompt的【学生背景】段落

### R9. 班级消息推送（P1）

**目标**：允许老师向班级或指定学生推送消息，消息直接显示在学生的聊天页，并通过微信服务通知触达提醒。

**核心功能**：
1. **消息推送**：
   - 教师选择目标（班级整体 / 指定学生） → 编辑消息内容 → 发送
   - 消息直接推送到学生与AI助手的聊天页，以系统消息形式展示
   - 学生打开聊天页即可看到教师推送的消息

2. **微信服务通知**：
   - 推送消息时同步发送微信订阅消息提醒
   - 学生点击通知跳转到对应聊天页面
   - 无需学生额外授权，复用现有订阅消息机制

3. **推送记录**：
   - 教师可查看推送历史（时间、目标、内容、送达状态）
   - 学生聊天页显示推送消息来源和发送时间
   - 记录推送状态（成功/失败）

**技术实现**：
- 数据库表：`teacher_messages`
  - 核心字段：`id`, `teacher_id`, `target_type`(class/student), `target_id`(class_id或student_id), `content`, `status`, `created_at`
- 聊天消息关联：推送消息写入`conversations`表，`role='system'`，`sender_type='teacher_push'`标注来源
- API：`POST /api/teacher-messages`、`GET /api/teacher-messages/history`
- 微信订阅消息：复用现有模板，跳转路径指向聊天页

**注意事项**：
- 推送频率限制：每个教师每天最多推送20条
- 单次推送人数上限100人
- 学生端聊天页需区分显示"教师推送消息"和"AI回复消息"

### R10. 记忆合并 - 用户画像持久化（P0）

**目标**：在记忆合并时，从对话历史中提炼用户画像（学生/老师），持久化存储，用于后续个性化服务。

**核心逻辑**：
1. **触发时机**：每次记忆合并时（现有记忆合并流程中增加一步）
2. **提炼内容**：
   - 学生画像：学习风格偏好、知识薄弱点、兴趣领域、性格特点、学习习惯等
   - 老师画像：教学风格偏好、关注重点、沟通习惯等
3. **存储方式**：每次提炼覆盖前一次，保持最新状态（非追加）
4. **使用场景**：注入到对话Prompt的【用户画像】段落，提升个性化服务精准度

**技术实现**：
- 数据库：`users`表增加`profile_snapshot`(JSON)字段，存储记忆合并时提炼的用户画像（学生/教师共用，通过`role`区分）
- 记忆合并流程修改：在`memory.MergeAndSummarize()`中增加画像提炼步骤
- LLM Prompt：要求从记忆中提取结构化画像信息，输出JSON格式

### R11. 语音输入支持（P1）

**目标**：在聊天页支持语音输入，提升交互便捷性，特别适合移动端和低年级学生。

**核心功能**：
1. **语音录制**：
   - 聊天输入框右侧增加语音按钮（麦克风图标）
   - 长按录音，松开发送；点击可切换为语音模式
   - 录制时长限制：最长60秒

2. **语音转文字**：
   - 使用微信小程序原生API `wx.startRecord` + `wx.getRecorderManager`
   - 调用语音识别服务（微信同声传译插件 或 云服务ASR）
   - 转换结果填充到输入框，用户可编辑后发送

3. **交互体验**：
   - 录音时显示音量波形动画
   - 支持取消录音（上滑取消）
   - 转换中显示loading状态

**技术实现**：
- 前端：聊天页组件增加语音录制逻辑
- ASR服务：优先使用微信同声传译插件，备选阿里云/腾讯云ASR
- API：`POST /api/speech-to-text`（如需服务端处理）

### R12. Emoji表情输入（P2）

**目标**：在聊天页右侧增加Emoji表情面板，丰富用户表达方式。

**核心功能**：
1. **表情面板**：
   - 输入框右侧增加表情按钮（😊图标）
   - 点击展开表情选择面板
   - 支持常用表情分类（笑脸、手势、动物、食物等）

2. **表情输入**：
   - 点击表情直接插入到输入框
   - 支持最近使用表情记录
   - 表情与文字混排

3. **交互细节**：
   - 面板高度适中，不遮挡聊天内容
   - 支持滑动切换表情分类
   - 发送后自动收起面板

**技术实现**：
- 前端：使用Taro组件库的Emoji面板或自定义实现
- 表情数据：内置常用Emoji列表（Unicode）
- 存储本地：最近使用表情存入本地Storage

## 验收标准

**R1 - 年级/教材配置**：教师选择年级即可完成适配；8档学段Prompt模板生效且优先级正确；支持全科+多教材+成人课程

**R2 - Adaptive RAG**：前沿/时事问题可触发外部搜索；搜索频率可控

**R3 - 用户反馈**：可提交多类型反馈；管理后台可查看和更新状态

**R6 - API安全**：全局限流 + 对话接口单独限流 + 文件上传校验

**R7 - 隐私防护**：学生无法通过prompt injection获取教师评价；LLM遇绕开指令保持角色不变

**R11 - 语音输入**：聊天页支持语音录制；语音转文字准确；支持取消和编辑

**R12 - Emoji表情**：表情面板正常展开；表情可插入输入框；最近使用记录生效
